# 🦟 Malaria Finder — Clinical Severity Assessment System

A machine-learning-powered web application that combines **clinical symptom analysis** with **deep-learning cell-image classification** to deliver a comprehensive malaria assessment toolkit. Built with **Streamlit**, **scikit-learn**, and **PyTorch**.

---

## 📋 Overview

Malaria Finder provides two complementary analysis modes:

1. **Clinical Symptom Assessment** — A Random Forest model trained on 30,000 patient records predicts malaria severity (Mild / Moderate / Severe) based on symptoms and demographics.
2. **Cell Image Analysis** — A ResNet-18 deep-learning model classifies thin blood-smear microscopy images as Parasitized or Uninfected.

Both modes return confidence scores and actionable clinical recommendations.

## 🧪 Model Training

### 1. Clinical Symptom Model (in-app)

The symptom-based Random Forest classifier trains **automatically** when the app starts. It uses `data/malaria_dataset.csv` with these features:

| Feature    | Type         | Description                           |
|------------|--------------|---------------------------------------|
| Fever      | Binary (0/1) | Whether the patient has fever         |
| Headache   | Binary (0/1) | Whether the patient reports headaches |
| Chills     | Binary (0/1) | Whether the patient experiences chills|
| Fatigue    | Binary (0/1) | Whether the patient shows fatigue     |
| Gender     | Categorical  | Male / Female                         |
| Age        | Numeric      | Patient age in years                  |
| Weight     | Numeric      | Patient weight in kg                  |
| Location   | Categorical  | Urban / Rural                         |
| **Severity** | **Target** | **Mild / Moderate / Severe**          |

### 2. Cell Image Model (ResNet-18)

The image classifier uses a fine-tuned **ResNet-18** architecture trained on the cell images dataset.

**Option A — Train locally:**

```bash
python train.py
```

This reads images from `data/cell_images/` (Parasitized & Uninfected folders) and saves the trained weights to `malaria_resnet18.pth`.

**Option B — Train on Google Colab:**

1. Upload `Clinic-Malaria.ipynb` to [Google Colab](https://colab.research.google.com/).
2. Upload the dataset (or mount Google Drive).
3. Run all cells to perform EDA, train, and evaluate.

### Cell Image Data Setup

```bash
kaggle datasets download -d iarunava/cell-images-for-detecting-malaria --unzip -p ./data
```

## 🚀 Running the Web App

### Prerequisites

- Python 3.9+
- pip

### Installation

```bash
# Clone the repository
git clone https://github.com/<your-username>/Clinical-System-For-Malaria-Severity-Assessment.git
cd Clinical-System-For-Malaria-Severity-Assessment

# Create and activate a virtual environment
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # macOS / Linux

# Install dependencies
pip install -r requirements.txt
```

### Launch

```bash
streamlit run app.py
```

The app will open at **http://localhost:8501**.
